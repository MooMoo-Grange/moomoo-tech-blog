# 예수원 홈페이지 전면 개편 — UX/UI 아키텍처

> "광야에 식탁을 차리실 수 있는가?" — 시편 78:19
>
> 이 문서는 예수원(Jesus Abbey) 홈페이지를 **숙박시설 안내 사이트**에서
> **오프라인 수도원 경험의 디지털 확장**으로 전환하기 위한 설계 아키텍처입니다.

---

## 1. 설계 철학 — 블루보틀 원칙의 수도원 적용

### 1.1 블루보틀이 한 것

블루보틀 커피는 매장의 핵심 경험을 추출하여 온라인에 이식했습니다.

| 오프라인 매장 | 온라인 번역 |
|---|---|
| 바리스타가 취향을 물어봄 | 커피 퀴즈로 개인 추천 |
| 천천히 핸드드립하는 시간 | 원두 산지 스토리, 느린 스크롤 |
| 미니멀한 공간, 커피에 집중 | 극도로 절제된 UI, 여백 중심 |
| 매장마다 다른 로컬 분위기 | 지역별 매장 스토리 페이지 |

블루보틀의 원칙: **"매장에서 3분간 느끼는 것을 온라인에서도 3분 안에 느끼게 하라."**

### 1.2 예수원이 해야 할 것

예수원의 오프라인 핵심 경험을 추출하고, 각각의 디지털 번역을 정의합니다.

| 오프라인 경험 | 감각/감정 | 디지털 번역 |
|---|---|---|
| 덕항산 기슭 도착, 산길을 오름 | 세상과의 분리, 고도 상승 | 진입 시퀀스: 느린 페이드인, 산 풍경 파노라마, 소리 제거 |
| 성당 종소리 | 시간의 구조화, 부름 | 시간 기반 UI: 접속 시각에 따른 기도 시간 안내 |
| 조도(새벽 기도) | 고요, 어둠 속의 빛 | 새벽 접속 시 다크 모드 + 촛불 톤 + 짧은 묵상 |
| 공동 노동(밭, 목장) | 땅의 감촉, 계절의 리듬 | 계절별 콘텐츠: 봄 파종, 여름 착유, 가을 수확, 겨울 기도 |
| 침묵 식사 | 말 없는 공유, 감사 | 최소 텍스트 섹션, 음식 사진 대신 식탁의 고요함 |
| 대천덕 신부 강의 테이프 | 권위 있는 가르침, 시간을 초월한 목소리 | 오디오 아카이브: 육성 강의 스트리밍 |
| 저녁 만도(기도) | 하루의 봉헌, 안식 | 저녁 접속 시 웜톤 + 저녁 기도문 + "오늘 하루를 봉헌합니다" |
| 덕항산 풍경 | 경외감, 하나님의 창조 | 풀스크린 풍경 사진, 계절/시간별 교체 |

**예수원의 원칙: "덕항산 기슭에 도착해서 첫 종소리를 듣기까지의 경험을 온라인에서 재현하라."**

## 2. 경험 설계 — 오프라인 여정의 디지털 번역

### 2.1 사용자 여정 맵 (User Journey)

예수원 방문자의 오프라인 여정을 5단계로 나누고, 각 단계의 디지털 대응을 설계합니다.

```
[오프라인 여정]

1. 부름 (Calling)     — "예수원이란 곳이 있대"
2. 여정 (Journey)     — 서울에서 태백까지 4시간, 산길을 오름
3. 진입 (Threshold)   — 예수원 문을 들어섬, 첫 종소리
4. 리듬 (Rhythm)      — 기도-노동-독서의 하루를 살아감
5. 파송 (Sending)     — 다시 세상으로, 그러나 변화된 채로
```

```
[디지털 여정]

1. 발견 (Discovery)   — 검색/SNS로 사이트 진입
2. 몰입 (Immersion)   — 히어로 시퀀스, 세상과의 분리 경험
3. 만남 (Encounter)   — 예수원의 이야기와 사람들을 만남
4. 체험 (Experience)  — 시간 기반 콘텐츠, 기도/묵상 참여
5. 응답 (Response)    — 방문 신청, 기도 요청, 후원, 구독
```

### 2.2 핵심 설계 원칙 (Design Principles)

**원칙 1: 시간이 구조다 (Time is Structure)**

수도원의 본질은 공간이 아니라 **시간의 구조화**입니다.
베네딕트 규칙서는 건물 설계가 아니라 시간표입니다.
웹사이트도 페이지 구조가 아니라 **시간 구조**로 설계합니다.

- 사용자의 접속 시각을 감지하여 UI 톤이 변합니다.
- 06:00-07:00 → 조도(새벽 기도) 모드: 깊은 남색 배경, 촛불 색 텍스트
- 12:00-12:30 → 대도(정오 기도) 모드: 밝은 자연광 톤, 노동 중단의 고요
- 19:30-20:30 → 만도(저녁 기도) 모드: 웜 앰버 톤, 저녁 기도문
- 그 외 시간 → 일상 모드: 현재의 abbey brown + forest green

**원칙 2: 여백이 침묵이다 (Whitespace is Silence)**

예수원에서 가장 강력한 경험 중 하나는 침묵입니다.
디지털에서 침묵은 **여백(whitespace)**입니다.

- 현재 사이트는 정보가 빽빽합니다. 모든 섹션 사이에 충분한 여백을 둡니다.
- 한 화면에 하나의 메시지만 전달합니다.
- 이모지, 뱃지, 태그 등 시각적 노이즈를 제거합니다.
- 스크롤 속도 자체가 느려지는 느낌을 줍니다 (패럴랙스가 아니라, 콘텐츠 밀도 조절).

**원칙 3: 서사가 설명을 대체한다 (Narrative over Explanation)**

현재 사이트는 예수원을 **설명**합니다 ("기도, 노동, 공동체의 삶을 통해...").
개편된 사이트는 예수원을 **이야기**합니다.

- "기도 생활" → "새벽 4시 반, 아직 어두운 성당에 촛불 하나가 켜진다"
- "노동 생활" → "얼어붙은 물통을 깨고 젖소에게 건초를 나른다"
- "공동체" → "말 없이 함께 앉은 식탁, 감사 기도 후 첫 숟가락"

모든 섹션이 추상적 설명이 아니라 **구체적 장면**으로 시작합니다.

**원칙 4: 점진적 노출 (Progressive Disclosure)**

수도원은 모든 것을 한꺼번에 보여주지 않습니다.
먼저 문을 들어서고, 복도를 걷고, 성당에 앉고, 그제서야 기도가 시작됩니다.

- 첫 화면: 풍경과 한 줄의 말씀만
- 스크롤: 예수원의 하루가 천천히 펼쳐짐
- 더 깊이: 설립자의 이야기, 60년의 역사
- 가장 깊이: 방문 신청, 공동체 참여

**원칙 5: 소리는 선택이다 (Sound is Optional but Sacred)**

수도원에서 소리는 의미를 가집니다 — 종소리, 찬송, 침묵 속 새소리.

- 사이트 진입 시 기본은 무음입니다.
- 사용자가 원하면 배경 소리를 켤 수 있습니다: 덕항산 자연음, 성당 종소리.
- 대천덕 신부님 육성 강의는 별도의 오디오 플레이어로 제공합니다.
- 소리를 강제하지 않습니다. 침묵도 경험입니다.

## 3. 정보 아키텍처 (Information Architecture)

### 3.1 현재 구조의 문제

```
[현재 — 시설 안내 패턴]

홈
├── 예수원 소개 (About)
├── 공동체 생활 (Features)
├── 설립자와 유산 (History)
├── 양성 및 교육 (Programs)
├── 방문 안내 (Booking)         ← 숙박시설 패턴
├── 출판 및 보급 (Shop)
└── 소식과 공동체 (News)
```

이 구조는 기업/시설 웹사이트의 전형적인 패턴입니다.
"우리 소개 → 서비스 → 이용 안내"의 흐름이 수도원의 경험과 맞지 않습니다.

### 3.2 제안하는 구조 — 경험 기반 아키텍처

```
[제안 — 경험 기반 패턴]

홈 (살아 있는 시간)
│
├── 예수원의 하루 (The Day)
│   ├── 새벽 — 기도 (Prayer)
│   ├── 아침 — 노동 (Labor)
│   ├── 정오 — 기도 (Prayer)
│   ├── 오후 — 독서와 침묵 (Reading & Silence)
│   ├── 저녁 — 기도 (Prayer)
│   └── 밤 — 고요 (Rest)
│
├── 예수원의 이야기 (The Story)
│   ├── 광야의 부름 — 설립 이야기 (1965~)
│   ├── 분수령 위의 삶 — 60년의 서사
│   ├── 사람들 — 대천덕, 현재인, 그리고 공동체
│   └── 예수원 이야기 블로그 (Notion CMS)
│
├── 예수원의 소리 (The Voice)
│   ├── 대천덕 신부 강의 아카이브
│   ├── 묵상과 기도문
│   └── 출판물 (광야의 식탁, 등)
│
├── 삼수령 무무목장 (The Grange)
│   ├── 목장의 하루
│   ├── A2 저지 젖소 이야기
│   └── 기도하며 일하라 (Ora et Labora)
│
├── 이 리듬에 함께하기 (Join the Rhythm)
│   ├── 방문하기 — 준비할 것들
│   ├── 수련 지원 — 공동체에 참여하기
│   ├── 기도로 함께하기 — 기도 요청/후원
│   └── 연락하기
│
└── 오늘의 묵상 (Daily Word)
    └── 매일 갱신되는 짧은 말씀/기도문
```

### 3.3 네비게이션 설계

```
[글로벌 네비게이션 — 최상단]

┌─────────────────────────────────────────────────────┐
│  ✝ 예수원                                            │
│                                                      │
│  예수원의 하루  ·  이야기  ·  소리  ·  무무목장       │
│                                        함께하기  →   │
└─────────────────────────────────────────────────────┘

- 메인 네비: 최대 5개 항목 (인지 부하 최소화)
- "함께하기"는 CTA로 분리 (방문/수련/기도/연락)
- 로고는 십자가 + "예수원" 텍스트 (이모지 아님, 실제 아이콘)
- 네비게이션 배경: 스크롤 시 반투명 → 불투명 전환
```

## 4. 페이지별 상세 설계

### 4.1 홈페이지 — "살아 있는 시간"

홈페이지는 **시간 감응형(Time-Responsive)**으로 설계합니다.

```
┌─────────────────────────────────────────────┐
│                                             │
│        [시간대별 풍경 사진 — 풀스크린]        │
│                                             │
│     "지금 예수원에서는                       │
│      저녁 기도가 시작되었습니다"              │
│                                             │
│     · 시편 134:1 ·                          │
│     "밤에 여호와의 성전에 서 있는             │
│      여호와의 모든 종들아                     │
│      여호와를 송축하라"                       │
│                                             │
│              ↓ 천천히 스크롤                  │
│                                             │
└─────────────────────────────────────────────┘
```

**시간대별 히어로 변화:**

| 시간 | 배경 | 텍스트 | 색조 |
|---|---|---|---|
| 05:00-06:59 | 덕항산 새벽, 안개 | "새벽이 오기 전, 기도가 시작됩니다" | 딥 네이비 + 촛불 앰버 |
| 07:00-11:59 | 목장/밭 노동 장면 | "손으로 기도하는 시간" | 내추럴 그린 + 브라운 |
| 12:00-12:59 | 성당 내부 | "잠시 멈추고, 하나님 앞에 섭니다" | 웜 골드 + 화이트 |
| 13:00-17:59 | 도서관/숲길 | "읽고, 걷고, 침묵합니다" | 소프트 세피아 |
| 18:00-19:29 | 공동 식탁 | "감사의 식탁에 함께 앉습니다" | 웜 앰버 |
| 19:30-21:59 | 성당 촛불 | "저녁 기도가 시작되었습니다" | 다크 웜 + 촛불 |
| 22:00-04:59 | 별이 빛나는 덕항산 | "고요 속에 하나님이 계십니다" | 다크 블루 + 별빛 |

**스크롤 시퀀스 (아래로 내리면):**

```
Section 1: 히어로 (시간 감응형)
    ↓
Section 2: "예수원의 하루" — 타임라인 미리보기
    하루를 6개 시간대로 나눈 세로 타임라인
    각 시간대에 사진 1장 + 한 줄 문장
    → "하루 전체 보기" 링크
    ↓
Section 3: "60년의 이야기" — 서사 프리뷰
    1965년 한 줄 → 2025년 한 줄
    타임라인이 아니라 "하나의 문장이 60년을 관통"
    예: "1965년, 한 사람이 산에 올랐습니다.
         60년이 지난 오늘, 그 산에는 여전히 종소리가 울립니다."
    → "이야기 전체 보기" 링크
    ↓
Section 4: "오늘의 묵상" — Daily Word
    매일 바뀌는 짧은 묵상 (Notion CMS 연동)
    성경 구절 + 1-2문장 묵상
    ↓
Section 5: "이 리듬에 함께하기"
    방문 / 기도 / 후원 — 3가지 참여 경로
    "예약하기"가 아니라 "이 리듬에 참여하기"
    ↓
Footer: 연락처, 위치, SNS
    교통 안내는 footer 또는 "함께하기" 하위에
```

### 4.2 "예수원의 하루" 페이지 — 리듬 체험

이 페이지는 사이트의 **심장**입니다.
사용자가 예수원의 하루를 스크롤로 따라가며 체험합니다.

```
구조: 세로 타임라인

05:00 ──── [새벽, 아직 어둡다]
            "종소리가 어둠을 깨운다.
             기도처로 향하는 발소리만이
             고요를 채운다."

06:00 ──── [조도 — 아침 기도]
            기도문 발췌 / 찬송 가사
            (선택: 실제 조도 녹음 재생)

07:00 ──── [아침 식사]
            "감사 기도 후 첫 숟가락.
             말이 없어도 함께라는 것을 안다."

08:00 ──── [노동 — 밭, 목장, 목공]
            사계절 노동 사진 슬라이드
            "얼어붙은 물통을 깨고
             건초를 나르는 손이
             기도하는 손과 같은 손이다."

12:00 ──── [대도 — 정오 기도]
            "삽을 내려놓고 성당으로 간다.
             흙 묻은 손으로 성경을 펼친다."

12:40 ──── [점심 식사]

14:00 ──── [독서와 침묵 — Lectio Divina]
            "읽는다. 반복한다. 묵상한다. 기도한다.
             네 단계가 하나의 호흡이 될 때까지."

17:00 ──── [자유 시간 / 산책]
            덕항산 숲길 사진

18:00 ──── [저녁 식사]

19:30 ──── [만도 — 저녁 기도]
            기도문 발췌
            "하루를 봉헌합니다.
             내일은 또 종소리로 시작될 것입니다."

22:00 ──── [고요]
            화면이 서서히 어두워짐
            "침묵 속에 하나님이 계십니다."
```

**기술 구현 노트:**
- Intersection Observer로 각 시간대 섹션 진입 시 배경색/톤 전환
- 각 시간대에 optional 오디오 버튼 (종소리, 새소리, 찬송)
- 모바일: 세로 스크롤 자연스럽게 유지, 사진은 풀폭
- 데스크톱: 왼쪽 시간 표시 고정, 오른쪽 콘텐츠 스크롤

### 4.5 "함께하기" 페이지 — 응답

**"방문 안내"를 "함께하기"로 리프레이밍합니다.**

제안: "이 리듬에 참여하는 방법이 있습니다" → 영적 초대

```

    "예수원의 하루를 직접 살아보세요"
    - 준비할 것 (마음의 준비 + 실제 준비물)
    - 하루 일과 미리보기
    - 신청 방법
    - 교통 안내
    ※ "숙소"가 아니라 "공동체의 일과에 참여"로 프레이밍

    "예수원에 오지 않아도 함께 기도할 수 있습니다"
    - 매일 기도 시간에 함께 기도하기
    - 기도 요청 보내기
    - 기도 후원 (재정 후원)

수련 지원 (Stay)
    "더 깊이, 더 오래 머물고 싶다면"
    - 장기 수련 안내
    - 공동체 참여 과정

연락하기 (Connect)
    전화, 이메일, 위치
```
